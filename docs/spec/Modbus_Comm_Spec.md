Modbus는 산업용 장비끼리 데이터를 주고받기 위한 통신 규약입니다.
쉽게 말하면 PLC, 온도조절기, 인버터, 전력계, 냉동기 컨트롤러 같은 장비에서 현재 온도, 압력, 운전 상태, 알람 상태, 설정값 등을 읽거나 쓰기 위해 많이 쓰는 방식입니다.

1. Modbus의 기본 개념

Modbus는 기본적으로 Master / Slave 구조입니다.

요즘 표현으로는 보통:

Client = 요청하는 쪽
Server = 응답하는 장비

라고도 합니다.

예를 들어 중앙 모니터링 PC가 여러 장비의 온도를 읽는다면,

모니터링 PC → “1번 장비야, 현재 온도 알려줘”
1번 장비 → “현재 온도는 23.5도야”

이런 식입니다.

2. Modbus의 대표 종류
   ① Modbus RTU

RS-485 시리얼 통신에서 가장 많이 사용합니다.

PC / PLC / 게이트웨이
│ RS-485
├── 장비 1
├── 장비 2
└── 장비 3

특징은:

2선식 RS-485에서 많이 사용
장비마다 국번 Slave ID가 있음
보통 1번, 2번, 3번처럼 주소를 정함
한 번에 하나의 장비와 통신
순차적으로 읽는 방식

예를 들어:

1번 장비 읽기 → 응답
2번 장비 읽기 → 응답
3번 장비 읽기 → 응답
다시 1번 장비 읽기...

이런 구조입니다.

② Modbus TCP

이더넷, 즉 LAN 통신에서 사용합니다.

모니터링 서버
│ LAN
├── 192.168.0.101 장비
├── 192.168.0.102 장비
└── 192.168.0.103 장비

특징은:

TCP/IP 기반
기본 포트는 502번
장비를 IP 주소로 구분
RS-485보다 빠르고 구성하기 쉬움
여러 장비에 동시에 접속할 수도 있음
그래도 프로그램 구조상 안정성을 위해 순차 polling을 많이 사용함

3. Modbus에서 데이터는 “레지스터”로 읽는다

Modbus 장비는 데이터를 보통 레지스터 주소에 저장해 둡니다.

예를 들어 냉동기 컨트롤러가 있다고 하면:

주소 의미 값
40001 현재 온도 235
40002 현재 압력 102
40003 운전 상태 1
40004 알람 코드 0

여기서 현재 온도 235는 실제로는 **23.5℃**일 수 있습니다.
즉, 장비 매뉴얼에서 “Scale 값”을 확인해야 합니다.

읽은 값: 235
스케일: ÷10
실제 값: 23.5℃ 4. Modbus 데이터 영역

Modbus에는 대표적으로 4가지 데이터 영역이 있습니다.

영역 용도 읽기/쓰기
Coil ON/OFF 출력 읽기/쓰기 가능
Discrete Input ON/OFF 입력 읽기 전용
Input Register 센서값 등 읽기 전용
Holding Register 설정값, 상태값 등 읽기/쓰기 가능

실무에서는 Holding Register를 가장 많이 봅니다.

예를 들어:

현재 온도 읽기
운전 상태 읽기
설정 온도 쓰기
기동/정지 명령 쓰기

이런 것들이 Holding Register에 들어가는 경우가 많습니다.

5. Function Code

Modbus 요청에는 “무슨 작업을 할지”를 나타내는 코드가 있습니다.

자주 쓰는 것은 이 정도입니다.

Function Code 의미
01 Coil 읽기
02 Discrete Input 읽기
03 Holding Register 읽기
04 Input Register 읽기
05 Coil 1개 쓰기
06 Holding Register 1개 쓰기
15 Coil 여러 개 쓰기
16 Holding Register 여러 개 쓰기

가장 많이 쓰는 건 03번, 04번, 06번, 16번입니다.

냉동기 모니터링 시스템에서 Modbus를 쓴다면 보통 이런 식입니다.
중앙 서버 → 냉동기 컨트롤러의 Holding Register 읽기

40001 현재 온도
40002 토출 압력
40003 흡입 압력
40004 운전 상태
40005 경보 상태
40006 설정 온도

서버는 일정 주기로 장비들을 읽습니다.

1초마다:
냉동기 1호기 데이터 읽기
냉동기 2호기 데이터 읽기
냉동기 3호기 데이터 읽기
...

6. Smart Web Router 사양

- Modbus 최대 한번에 읽을 수 있는 word 갯수를 32개로 제한한다. (특정 제품은 한번에 읽을수 있는 제한이 있는 경우가 있으므로) 만약 32 이상의 요청이 있는 경우 적절한 오류 코드로
  응답한다.
